# Security Policy

## Supported Versions

We take the security of vigilcd seriously. To ensure the best protection for our users, we only support security updates for the latest major version.

| Version | Supported          |
| ------- | ------------------ |
| latest   | :white_check_mark: |
| < latest   | :x:              |


We recommend always running the most recent release to benefit from the latest security patches and features.

## Reporting a Vulnerability

If you discover a security vulnerability within this project, please do not open a public issue. This allows us to fix the flaw before it is exploited.

### How to report

Please report vulnerabilities via one of the following methods:

- GitHub Security Advisory: Use the "Report a vulnerability" button under the Security tab of this repository (preferred).
- Github Message or LinkedIn Message to @ruben-sprengel

### What to include

To help us triage the issue quickly, please include:

- A descriptive title.
- Steps to reproduce the issue (a Proof of Concept script or detailed description).
- Potential impact (e.g., Remote Code Execution, Information Disclosure).

### Our Process

- Acknowledgement: You will receive an acknowledgement of your report within 48 hours.
- Triage: We will investigate the report and keep you informed of our progress.
- Fix: If the vulnerability is confirmed, we will work on a fix and release a new version.

Thank you for helping keep this project secure!
